# 🛡️ PhishGuard AI — Phishing Website Detector

[**🌐 Live Demo Dashboard**](https://phishguard-ai-q4l9.onrender.com)

**PhishGuard AI** is a full-stack, production-ready phishing website detection system powered by an ensemble of machine learning models. It combines a high-accuracy AI backend with a real-time, professional-grade **Quantum Security Dashboard** built on Flask and modern CSS/SVG animations.

> Built for academic research and real-world security analysis. Detects phishing URLs with high confidence using structural and lexical feature extraction.

---

## 🚀 Key Features

- **AI Ensemble Classifier** — Combines Random Forest, XGBoost, LightGBM, and Logistic Regression via soft-voting for superior accuracy.
- **LSTM Benchmark** — A PyTorch-based LSTM model is included for performance comparison.
- **Real-Time Inference** — Analyze any URL instantly through the browser-based dashboard.
- **Quantum Security Dashboard** — A pixel-perfect, animated UI with:
  - SVG Circular Gauge with "round motion" draw-in animations.
  - Glassmorphism cards with neon accent stripes.
  - Real-time Lexical Strength, Host Reliability, and Global Threat Score metrics.
  - Color-coded threat states (Cyan for SAFE, Red for PHISHING).
- **SHAP Interpretability** — SHAP value visualizations for model explainability.
- **Ablation Study** — Systematic analysis of individual feature contributions.

## 🧠 AI Architecture

### Ensemble Model
The primary classifier uses a `VotingClassifier` (soft-voting) combining:

| Model | Role |
|---|---|
| Random Forest | Captures non-linear feature interactions |
| XGBoost | Gradient-boosted tree ensemble for high accuracy |
| LightGBM | Fast, memory-efficient gradient boosting |
| Logistic Regression | Linear baseline for calibration |

The final prediction is derived from the **averaged class probabilities** of all four models, making the system more robust and less likely to overfit.

### LSTM Benchmark (`lstm_benchmark.py`)
A PyTorch-based sequential LSTM model trained on the same feature set to benchmark performance against the ensemble approach.

## 🔬 Feature Engineering

The system extracts **14 structural and lexical features** from each URL using `src/features.py`:

| # | Feature | Description |
|---|---|---|
| 1 | `url_length` | Total character length of the URL |
| 2 | `num_dots` | Count of `.` separators (subdomains) |
| 3 | `num_hyphen` | Count of `-` characters |
| 4 | `num_at` | Count of `@` (common in phishing) |
| 5 | `num_question` | Count of `?` (query strings) |
| 6 | `num_equal` | Count of `=` (parameter assignments) |
| 7 | `num_and` | Count of `&` (chained parameters) |
| 8 | `num_percent` | Count of `%` (URL encoding) |
| 9 | `num_slash` | Count of `/` (path depth) |
| 10 | `num_www` | Presence of `www` subdomain |
| 11 | `num_http` | Count of `http` occurrences |
| 12 | `num_https` | Count of `https` occurrences |
| 13 | `has_ip` | Whether an IP address is used instead of a domain |
| 14 | `path_length` | Length of the URL path component |

## 📊 Real-Time Dashboard Metrics

When a URL is analyzed, the dashboard computes and displays:

| Metric | Description |
|---|---|
| **Security Assessment** | Overall AI confidence score (0–100) from the ensemble's `predict_proba` |
| **Lexical Strength** | URL structure quality score (penalizes length, hyphens, special chars) |
| **Host Reliability** | Protocol and domain trust score (rewards HTTPS, penalizes IPs) |
| **Global Threat Score** | Scaled threat intensity directly from the AI probability output |
| **Hostname** | Extracted base domain of the input URL |
| **Protocol** | HTTP vs. HTTPS detection |
| **Anomalies** | Count of suspicious special characters (`@`, `%`, `&`, `=`) |
| **Analysis Engine** | Identifies the AI model used: `Ensemble AI (RF+XGB+LGBM)` |

## ⚙️ Setup & Installation

### Prerequisites
- Python 3.8+
- pip

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd Phishing-Website-Detector-using-LightGBM-XGBoost-Random-Forest-and-Logistic-Regression
```

### 2. Install Dependencies

```bash
pip install -r requirements.txt
```

**Dependencies include:**
```
flask, pandas, numpy, scikit-learn, xgboost, lightgbm, joblib, torch, shap, matplotlib
```

### 3. Train the Model

Run the training pipeline to generate the ensemble model:

```bash
python app.py
```

This will:
- Load and preprocess `url_dataset.csv`
- Train the VotingClassifier ensemble
- Save the model to `models/phishing_ensemble_model.pkl`
- Print accuracy, classification report, and confusion matrix

### 4. Launch the Dashboard

```bash
python web_app.py
```

Open your browser and navigate to:
```
http://127.0.0.1:5000
```

## 🖥️ Dashboard Usage

1. Enter any URL in the search bar (e.g., `https://www.google.com`).
2. Click **Analyze URL**.
3. The dashboard will display:
   - **Security Assessment Gauge** — Animated SVG arc drawing from 0 to the AI score.
   - **URL Details Card** — Hostname, Protocol, Status, Risk, Anomalies, and Engine.
   - **Lexical Strength** — Segmented bar chart with pulsing neon indicators.
   - **Host Reliability** — WHOIS processing status, domain age, and server trust.
   - **Global Threat Score** — Threat intensity with CLEAN/DANGEROUS badge.

## 📌 Known Limitations

- **Domain Age**: The `domain_age` value is inferred from the AI's prediction (not from a live WHOIS API). Integrating a WHOIS API (e.g., DomainTools) would provide exact registration dates.
- **Development Server**: The Flask development server is used for local testing. For production deployment, use a WSGI server such as **Gunicorn** or **Waitress**.
- **Feature Scope**: The 14 features are purely structural/lexical. Content-based and graph-based features could be added in future iterations to improve robustness.
